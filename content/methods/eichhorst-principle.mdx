## What it is

The Eichhorst Principle applies Claude Shannon's noisy-channel idea to AI coding. The large
language model is the noisy channel. It can guess right, drift, or fail. The feedback loop is what
makes the result reliable.

In practice, the loop is simple: generate code, compile it, run the tests, read the failures, and
try again. Each check catches a different error class. The compiler catches syntax and wiring
errors. Tests catch logic and behavior errors.

## Why it matters

The key rule is blunt: the loop can only fix what it can detect. That means better tests usually
matter more than better prompts. A strong test suite turns an unreliable generator into a much more
reliable coding partner.

This makes the Eichhorst Principle a useful lens for agentic development. If the agent keeps
breaking the same kind of behavior, the first question is not "how do I rewrite the prompt?" It is
"what check is missing from the loop?"
